Agora grid

A proportional and responsive grid system. You can use it to indicate which percentage of the screen each element should use for different screen sizes. It is nestable, and allows semantic classes.

Basic concepts

The grid system is based in two main elements identified by the following classes:

  • mw-ui-grid: The grid container. Layout properties work only for elements contained in a grid.
  • mw-ui-item: Elements inside a grid. Additional classes are normally addeed to customize the layout of the element.

Layout instructions are provided by classes with the following synthax: mw-ui[-medium][-set]-[proportion]. More details below.

History

The grid is based on the concepts defined by CSS Wizardry in this grid system. The former grid system uses inline-block elements which requires to avoid spaces between blocks. Agora grid uses (some of) the same concepts but relies on floating elements instead. A proportional grid system built by Boon has been extended to spport the concepts above, and made available in both Sass and Less.

Proportionally responsive

The mediums availabe define breaking points in a no-device specific way:

  • mw-ui-palm for small screens that fit your hand.
  • mw-ui-lap for medium screens.
  • mw-ui-portable for both of the above.
  • mw-ui-desk for bigger screens.
  • (default)

The proportions allow to work with columns in different fractions such as one-half, three-quarters or eleven-twelfths.

Regular containers

When all elements in a container have the same widths, you can indicate so at the grid element avoiding to repeat the definition at each element. The synthax for doing so is similar to the one used for items but uses the -set prefix.

Nestable

Grids can be defined inside other grids.

Semantic clases

The grid is available in both less and sass. So you can create your own classes by extending the existing ones.
